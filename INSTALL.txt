INSTALLATION / BUILD INSTRUCTIONS (from source)
=============================================

This project is a Python package (see `setup.py`). The instructions below install
all dependencies and install the package from source so you can run the server
and scripts.


0) Supported platforms
----------------------

- Linux is the primary target (these steps assume Ubuntu/Debian).
- Python version required: >= 3.8.1 (see `setup.py`).


1) System prerequisites (Ubuntu/Debian)
---------------------------------------

Install Python + build tooling:

```bash
sudo apt-get update
sudo apt-get install -y \
  python3 \
  python3-venv \
  python3-pip \
  python3-dev \
  build-essential \
  git \
  curl
```

If your distro’s default `python3` is older than 3.8, install a newer Python.
On Ubuntu, one common approach is Deadsnakes PPA:

- URL: https://launchpad.net/~deadsnakes/+archive/ubuntu/ppa

Example (Ubuntu):

```bash
sudo apt-get update
sudo apt-get install -y software-properties-common
sudo add-apt-repository -y ppa:deadsnakes/ppa
sudo apt-get update
sudo apt-get install -y python3.10 python3.10-venv python3.10-dev
```


2) Get the source code
----------------------

If you already have the repo, skip this step.

```bash
git clone https://github.com/zilliztech/GPTCache.git ContextCache
cd ContextCache
```

If you are working in a fork/branch, clone that repo URL instead.


3) Create and activate a virtual environment
--------------------------------------------

Using the system default `python3`:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -U pip setuptools wheel
```

If you installed a specific Python (example `python3.10`):

```bash
python3.10 -m venv .venv
source .venv/bin/activate
python -m pip install -U pip setuptools wheel
```


4) Install base dependencies and install the package (editable)
--------------------------------------------------------------

This installs dependencies from `requirements.txt` and installs the package from
your local checkout so code changes take effect immediately.

```bash
python -m pip install -r requirements.txt
python -m pip install -e .
```

Sanity check import:

```bash
python -c "import gptcache; print('gptcache version:', getattr(gptcache, '__version__', 'unknown'))"
```

4b) Build a wheel/sdist (non-editable “compile” step)
-----------------------------------------------------

If you need an actual build artifact (wheel/sdist) from the source tree:

- Python build docs URL: https://pypi.org/project/build/

```bash
python -m pip install -U build
python -m build
```

This creates `dist/*.whl` and `dist/*.tar.gz`. To install the wheel you built:

```bash
python -m pip install --force-reinstall dist/*.whl
```


5) Install runtime dependencies for the API server
--------------------------------------------------

The FastAPI server imports these at runtime (they are intentionally not pinned in
`requirements.txt` in this repo). Install them explicitly:

```bash
python -m pip install \
  fastapi \
  "uvicorn[standard]" \
  pydantic \
  starlette
```

Run the cache API server:

```bash
python -m gptcache_server.server -s 127.0.0.1 -p 8011 -d /tmp/contextcache_data
```

Health check:

```bash
curl -s http://127.0.0.1:8011/
```


6) Optional: OpenAI-compatible proxy mode dependencies
------------------------------------------------------

To use `POST /v1/chat/completions`, install the OpenAI Python SDK:

- URL: https://pypi.org/project/openai/

```bash
python -m pip install openai
```

Then run the proxy-enabled server:

```bash
env -u APPIMAGE /usr/bin/python3 -m gptcache_server.server \
  -s 127.0.0.1 -p 8012 \
  -d /tmp/contextcache_data \
  -o True
```

You will also need an OpenAI API key available to the client sending requests
(the server reads it from the `Authorization: Bearer ...` header as shown in
`README.md`).


7) Optional: FAISS vector index backend
---------------------------------------

FAISS is used by `gptcache/manager/vector_data/faiss.py`. Install the CPU wheel:

- URL: https://pypi.org/project/faiss-cpu/

```bash
python -m pip install faiss-cpu
```

If you want GPU FAISS, use `faiss-gpu` (availability depends on your platform):

- URL: https://pypi.org/project/faiss-gpu/


8) Optional: ONNXRuntime evaluator (and model download tooling)
--------------------------------------------------------------

The ONNX similarity evaluator uses:
- `onnxruntime`
- `huggingface_hub` (downloads `model.onnx`)
- `transformers` (already in `requirements.txt`)

Install:

- onnxruntime URL: https://pypi.org/project/onnxruntime/
- huggingface-hub URL: https://pypi.org/project/huggingface-hub/

```bash
python -m pip install onnxruntime huggingface-hub
```

Note: the evaluator downloads the ONNX model from Hugging Face at runtime
(default repo id: `GPTCache/albert-duplicate-onnx`).


9) Optional: PyTorch CUDA vs CPU wheels
---------------------------------------

`requirements.txt` includes `torch>=2.0`. By default, pip will typically install
a CPU wheel from PyPI. If you need a CUDA-enabled build, install following the
official selector:

- URL: https://pytorch.org/get-started/locally/

Example (CUDA wheels vary by version; follow the website for the exact command).


10) Uninstall / clean rebuild
-----------------------------

To remove the editable install:

```bash
python -m pip uninstall -y gptcache
```

To reinstall after changing dependencies:

```bash
python -m pip install -U pip setuptools wheel
python -m pip install -r requirements.txt
python -m pip install -e .
```
